# Input Environment seam

Instruction Mode should depend on an Input Environment seam instead of calling `typer` and `TextBuffer` details directly. The interface should expose domain operations such as reading the current Explicit Selection, discovering the current safe Operation Window, inserting text, revising or removing a verified Operation Target, replacing an Explicit Selection, and moving out of an Explicit Selection before insertion. This concentrates the "Explicit Selection first; no-selection local partial edits fail closed; whole-scope requests may use the current Operation Window" rule in one deep module while allowing platform-specific typing, clipboard, cursor, and tracking adapters to remain implementation details.
