# Input Environment seam

Instruction Mode should depend on an Input Environment seam instead of calling `typer`, `TextBuffer`, keyboard monitoring, and mouse monitoring details directly. The interface should expose domain operations such as reading the current Explicit Selection, discovering the current safe Operation Window, inserting text, revising or removing a verified Operation Target, replacing an Explicit Selection, moving out of an Explicit Selection before insertion, and reporting when fallback Tracked Segment state is unsafe. This concentrates the "Explicit Selection first, otherwise current safe Operation Window, then safe Tracked Segment fallback" rule in one deep module while allowing platform-specific typing, clipboard, cursor, and tracking adapters to remain implementation details.
